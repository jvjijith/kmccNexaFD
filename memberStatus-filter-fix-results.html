MemberStatus Filter Fix - Implementation Summary

üîç Problem Identified

ISSUE: The memberStatus filter was not working because the useGetData hook was not refetching when filter values changed.
Root Cause:
  • The useApiQuery hook was using a static queryKey that didn't include the URL
  • React Query only refetches when the queryKey changes
  • Since the queryKey was static, changing filters didn't trigger a refetch

‚úÖ 1. Fixed useApiQuery Hook

SUCCESS: Updated the queryKey to include the URL parameter
Location: src/common/api/index.jsx
Change: queryKey: [key, url] // Now includes URL in query key
Impact: Query will refetch automatically when URL changes

‚úÖ 2. Improved URL Building

SUCCESS: Enhanced the API URL building to be more dynamic
Location: src/layout/component/membershipTable.jsx
Change: const apiUrl = `/members?${buildQueryString()}`;
Impact: URL is rebuilt every time filters or currentPage changes

‚úÖ 3. Added Debug Logging

SUCCESS: Added console logging to help debug filter issues
console.log('API URL:', apiUrl);
console.log('Current filters:', filters);
How to use: Open browser dev tools and watch the console when changing filters to verify the URL is being updated correctly.

‚úÖ 4. Maintained usePutData Integration

SUCCESS: The usePutData hook integration for member updates remains intact
  • Accept/Reject functionality still uses usePutData hook
  • Loading states are preserved on buttons
  • Automatic refetch after member status updates

üîß Technical Details

Filter Options Available:
  • pending - Members awaiting approval
  • accepted - Approved members
  • rejected - Rejected applications
  • "" (empty) - All members (no filter)
API Endpoint: GET /members?memberStatus=pending&page=1&limit=10
Query Parameters: memberStatus, page, limit, applicationFor, paymentStatus, visaStatus, district, search
